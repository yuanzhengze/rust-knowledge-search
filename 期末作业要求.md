# 期末作业要求

# 作业目标

本课程期末大作业旨在帮助学生综合运用 Rust 语言知识，完成一个具有实际功能的完整项目。项目应体现：

- Rust 基础语法与类型系统

- 所有权、借用的合理使用

- 模块化设计

- 错误处理

- 工程化开发流程

- 文档与测试能力

---

# 项目形式

学生可以选择：

- 单人完成 

- 组队完成（3至5人） 

鼓励根据兴趣自由选题。

**请勿直接抄袭现成开源项目，评审时会做检查。**

---

# 项目选题要求

## 允许自由选题

项目主题不限，但必须满足：

- 主要代码使用 Rust 编写 

- 具有完整、可运行的功能 

- 体现 Rust 的核心特性 

- 项目复杂度达到课程要求 

---

## 不建议的项目类型

以下类型通常难以体现 Rust 特性，不建议选择：

- 纯前端页面项目 

- 简单 CRUD \(增删改查\) 管理系统 

- 仅调用第三方 API 的包装器 

- 主要逻辑由 AI 自动生成的项目 

- Rust 代码占比极低的项目 

---

## 鼓励的项目方向

鼓励尝试：

- 系统工具 

- 命令行工具 

- 网络服务 

- 并发程序 

- 编译器/解释器 

- 数据处理工具 

- TUI（Terminal UI） 

- 小型数据库 

- 文件系统工具 

- Rust 不常见方向的探索性项目 

特别鼓励使用 Rust 完成一些其他语言更常见、但较少有人用 Rust 实现的项目

例如：

- Markdown 实时协作工具 

- 本地知识库搜索系统 

- 音频处理工具 

- PDF 分析工具 

- 小型视频处理流水线 

- 浏览器插件后端 

- 本地 AI 推理工具链 

- Git 可视化分析工具 

- Rust 所有权可视化分析工具

- 代码静态分析工具

- 网络抓包分析器 

- 多线程爬虫系统 

- 自定义 shell 

---

# 项目规模要求

## 单人项目

推荐规模：

- 1500～3000 行有效 Rust 代码 

- 至少 3 个模块 

要求：

- 功能完整 

- 可以独立运行 

- 提供完整文档 

---

## 组队项目（3～5 人）

推荐规模：

- 3000～6000 行有效 Rust 代码 

- 明确模块划分 

- 体现协作开发过程 

额外要求：

- 使用 Git 协作 

- commit 历史清晰 

- 每位成员有明确贡献 

- 提供成员分工说明 

组队项目不意味着简单分工拼接，而应体现：

- 接口设计 

- 模块协作 

- 工程组织能力 

---

# 技术要求

项目必须体现 Rust 工程实践。

### 模块化设计

合理使用：

- mod 

- crate 

- workspace（大型项目推荐） 

---

### 错误处理

要求：

- 使用 Result 

- 合理传播错误 

- 尽量避免大量使用 unwrap/expect 规避错误处理 

---

### Rust 核心特性

项目中应体现：

- ownership / borrowing 

- struct / enum 

- trait 

- 泛型 

- 生命周期（如需要） 

---

### 并发或异步（如需要）

可选：

- thread 

- channel 

- mutex/rwlock 

- async/await 

- tokio 

---

### 测试

至少包含：

- 单元测试 

- 若干关键功能测试 

---

### 工程规范

提交前需通过：

- cargo fmt 

- cargo clippy 

---

### 项目文档（README）

包括使用方法、编译运行方式、依赖说明等

---
