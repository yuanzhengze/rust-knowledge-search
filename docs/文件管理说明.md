# 文件管理说明

本文档说明仓库中的文件用途，帮助成员快速找到需要维护的内容。

## 根目录文件

- `README.md`：项目入口文档，介绍项目、运行方式、成员分工和协作资料。
- `agent.md`：仓库级 Agent 协作规范，约束 AI 和成员的开发行为。
- `plan.md`：两周开发计划，包含每日任务和里程碑。
- `项目协作文档.md`：面向组员的协作说明，包含分工、接口、Git 和测试要求。
- `CONTRIBUTING.md`：贡献规范，说明分支、提交和合并前检查。
- `开发日志.md`：记录每日开发进展，后续写报告时使用。
- `.gitignore`：避免提交构建产物、缓存、密钥和本地临时文件。
- `.env.example`：环境变量模板，用于说明 AI API 配置项。

## docs 目录

- `docs/接口设计.md`：核心数据结构、模块接口和接口修改流程。
- `docs/测试计划.md`：单元测试、集成测试和手动测试计划。
- `docs/演示脚本.md`：5 分钟演示视频的讲解流程。
- `docs/文件管理说明.md`：当前文件用途说明。

## examples 目录

- `examples/sample_notes/`：示例知识库资料，用于开发测试和演示视频。

## 后续建议新增

Rust 项目初始化后建议新增：

- `Cargo.toml`：Rust 项目依赖配置。
- `Cargo.lock`：应用项目建议提交，保证依赖版本稳定。
- `src/`：Rust 源码目录。
- `tests/`：集成测试目录。

## 维护规则

- 改动命令行用法时，同步更新 `README.md` 和 `docs/演示脚本.md`。
- 改动公共数据结构时，同步更新 `docs/接口设计.md`。
- 改动测试方式时，同步更新 `docs/测试计划.md`。
- 改动成员分工时，同步更新 `README.md`、`项目协作文档.md` 和实验报告。
- 每次阶段性完成后，在 `开发日志.md` 中记录进展。
