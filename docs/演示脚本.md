# 演示视频脚本

本文档用于准备期末作业 5 分钟以内的功能演示视频。实际录制时可以根据最终实现情况微调。

## 1. 演示目标

演示需要让老师快速看到：

- 项目是一个完整可运行的 Rust 工具。
- 项目不是简单 API 包装，而是包含文件扫描、解析、索引、搜索等核心逻辑。
- 项目体现了 Rust 的模块化、错误处理、数据结构和工程实践。
- 小组成员分工明确。
- 项目具备一定创新性，支持本地知识库检索和可选 AI 语义增强。

## 2. 视频总时长

建议控制在 4 分 30 秒到 5 分钟之间。

## 3. 演示流程

### 0:00 到 0:40：项目背景与目标

讲解要点：

- 我们实现的是基于 Rust 的本地知识库语义搜索系统。
- 用户可以把课程笔记、Markdown 文档、TXT 资料放到本地目录中。
- 系统会自动扫描、解析并建立索引。
- 用户可以通过关键词或自然语言问题检索资料。
- AI 功能只是增强，核心检索流程由 Rust 实现。

示例讲稿：

```text
大家好，我们小组的 Rust 期末项目是“基于 Rust 的本地知识库语义搜索系统”。这个项目面向本地资料检索场景，用户可以指定一个资料目录，系统会扫描 Markdown 和 TXT 文档，构建索引，并支持关键词搜索和自然语言问答。我们重点使用 Rust 实现文件处理、索引构建、搜索排序和错误处理，AI 只作为增强能力。
```

### 0:40 到 1:30：项目结构与成员分工

展示内容：

- 打开项目目录。
- 展示 `src/` 模块结构。
- 展示 `README.md` 或 `项目协作文档.md` 中的分工。

讲解要点：

- `scanner` 负责文件扫描。
- `parser` 负责文档解析。
- `chunker` 负责文本切片。
- `indexer` 和 `search` 负责索引和搜索。
- `semantic` 负责语义增强。
- `cli` 负责命令行交互。

示例讲稿：

```text
项目采用模块化设计。scanner 模块负责递归扫描文档，parser 模块负责读取和清洗文本，chunker 模块把长文档切成片段，indexer 构建倒排索引，search 根据查询返回排序结果，semantic 模块提供可选的语义增强能力。每位成员都负责一个明确模块，并通过 Git 记录协作过程。
```

### 1:30 到 2:30：建立索引

演示命令：

```bash
cargo run -- index ./examples/sample_notes
```

展示内容：

- 示例资料目录。
- 运行索引命令。
- 输出扫描文件数量、切片数量、索引关键词数量。

讲解要点：

- 系统扫描本地文档。
- 解析 Markdown/TXT。
- 将文档切片。
- 构建索引缓存。

示例讲稿：

```text
这里我们使用 examples/sample_notes 作为示例知识库。运行 index 命令后，程序会扫描目录中的 Markdown 和 TXT 文件，解析内容并切分成多个文档片段，随后构建关键词倒排索引。索引结果可以缓存到本地，避免每次搜索都重新扫描。
```

### 2:30 到 3:30：关键词搜索

演示命令：

```bash
cargo run -- search "Rust"
cargo run -- search "所有权" --top 5
```

展示内容：

- 搜索结果路径。
- 文档片段。
- 相关度分数。
- 关键词高亮，如果已实现。

讲解要点：

- 搜索模块根据查询词找到候选片段。
- 使用词频、TF-IDF 或简化排序算法计算相关度。
- 返回 Top K 结果。

示例讲稿：

```text
接下来演示关键词搜索。输入 Rust 或所有权后，系统会从索引中查找相关片段，并根据相关度排序。每条结果会展示来源文件、文本片段和分数，方便用户快速定位原文。
```

### 3:30 到 4:30：自然语言问答或语义增强

演示命令：

```bash
cargo run -- ask "Rust 的所有权解决了什么问题？"
```

展示内容：

- 自然语言问题。
- 返回的相关文档片段。
- 可选 AI 总结回答。

讲解要点：

- `ask` 命令先检索相关片段。
- 如果 AI 可用，再基于片段生成简短回答。
- 如果 AI 不可用，也能回退到展示检索结果。

示例讲稿：

```text
ask 命令用于自然语言查询。系统会先检索与问题最相关的文档片段，如果配置了 AI 服务，就可以基于这些片段生成简短回答。如果没有配置 API Key，程序也不会失败，而是回退为展示相关资料片段。
```

### 4:30 到 5:00：Rust 特性与总结

讲解要点：

- 使用 `struct` 和 `enum` 建模文档、索引和错误。
- 使用 `Result` 进行错误处理。
- 使用 `HashMap` 构建倒排索引。
- 使用模块化设计组织工程。
- 使用测试保证核心功能正确。

示例讲稿：

```text
总结来说，本项目体现了 Rust 的模块化工程实践、所有权和借用管理、Result 错误处理、HashMap 等数据结构使用，以及测试和文档能力。相比单纯调用 AI API，我们把 Rust 作为核心检索系统的实现语言，AI 只是增强层。以上就是我们的项目演示。
```

## 4. 录制前检查清单

录制前确认：

- 示例资料目录存在。
- `cargo build` 可以通过。
- `cargo run -- index ./examples/sample_notes` 可以运行。
- `cargo run -- search "Rust"` 可以返回结果。
- `cargo run -- search "所有权" --top 5` 可以返回结果。
- `cargo run -- ask "Rust 的所有权解决了什么问题？"` 可以运行或合理降级。
- 终端字体足够大。
- 不展示 API Key、隐私路径或无关窗口。

## 5. 如果 AI 功能未完成

可以这样说明：

```text
语义增强模块在接口上已经预留，可以接入 embedding API 或本地模型。当前版本的 ask 命令会先基于关键词检索返回相关片段，保证在没有网络或 API Key 的情况下仍然可以完成本地知识库查询。
```
