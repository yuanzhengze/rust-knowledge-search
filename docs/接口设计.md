# 接口设计

本文档用于约定项目核心数据结构、模块职责和函数接口，避免多人开发时出现接口不一致。

## 1. 模块关系

推荐处理流程：

```text
CLI
  ↓
scanner 扫描文件
  ↓
parser 解析文档
  ↓
chunker 切分文本片段
  ↓
indexer 构建索引
  ↓
search 执行搜索和排序
  ↓
semantic 可选语义增强
  ↓
CLI 输出结果
```

## 2. 核心数据结构

### DocumentMeta

用于描述文件元数据。

```rust
pub struct DocumentMeta {
    pub path: PathBuf,
    pub file_name: String,
    pub file_size: u64,
    pub modified_time: Option<SystemTime>,
}
```

字段说明：

- `path`：文件完整路径。
- `file_name`：文件名。
- `file_size`：文件大小，单位为字节。
- `modified_time`：最后修改时间，读取失败时为 `None`。

### Document

用于表示解析后的完整文档。

```rust
pub struct Document {
    pub meta: DocumentMeta,
    pub title: Option<String>,
    pub content: String,
}
```

字段说明：

- `meta`：文档元数据。
- `title`：文档标题，Markdown 可取第一个一级标题。
- `content`：清洗后的正文。

### DocumentChunk

用于表示搜索和语义分析的最小文本片段。

```rust
pub struct DocumentChunk {
    pub id: String,
    pub file_path: PathBuf,
    pub title: Option<String>,
    pub content: String,
    pub start_line: usize,
    pub end_line: usize,
}
```

字段说明：

- `id`：片段唯一 ID。
- `file_path`：片段来源文件。
- `title`：来源文档标题。
- `content`：片段文本内容。
- `start_line`：片段起始行。
- `end_line`：片段结束行。

### SearchResult

用于表示搜索结果。

```rust
pub struct SearchResult {
    pub chunk_id: String,
    pub file_path: PathBuf,
    pub snippet: String,
    pub score: f64,
    pub matched_terms: Vec<String>,
}
```

字段说明：

- `chunk_id`：命中的文档片段 ID。
- `file_path`：命中文件路径。
- `snippet`：用于展示的摘要片段。
- `score`：相关度分数。
- `matched_terms`：匹配到的关键词。

## 3. 错误类型约定

建议在 `error.rs` 中定义统一错误类型。

```rust
#[derive(Debug, thiserror::Error)]
pub enum AppError {
    #[error("IO error: {0}")]
    Io(#[from] std::io::Error),

    #[error("invalid path: {0}")]
    InvalidPath(String),

    #[error("parse error: {0}")]
    Parse(String),

    #[error("index error: {0}")]
    Index(String),

    #[error("semantic error: {0}")]
    Semantic(String),
}

pub type AppResult<T> = Result<T, AppError>;
```

所有模块优先返回 `AppResult<T>`。

## 4. scanner 模块

职责：

- 递归扫描指定目录。
- 过滤支持的文件类型。
- 读取文件元数据。
- 返回文档元数据列表。

建议接口：

```rust
pub fn scan_documents(root: &Path) -> AppResult<Vec<DocumentMeta>>;
```

要求：

- 支持 `.md` 和 `.txt`。
- 忽略隐藏文件和目录。
- 对不存在路径返回错误。
- 不在 scanner 中读取完整文件内容。

## 5. parser 模块

职责：

- 读取文件内容。
- 清洗 Markdown/TXT 文本。
- 提取标题。
- 返回 `Document`。

建议接口：

```rust
pub fn parse_document(meta: DocumentMeta) -> AppResult<Document>;
```

要求：

- Markdown 可去除基础标记。
- TXT 直接读取正文。
- UTF-8 读取失败时返回清晰错误。

## 6. chunker 模块

职责：

- 将 `Document` 切分为多个 `DocumentChunk`。
- 保留来源文件和行号范围。

建议接口：

```rust
pub fn chunk_document(document: &Document, max_chars: usize) -> Vec<DocumentChunk>;
```

要求：

- 每个 chunk 应有稳定 ID。
- 内容不宜过短或过长。
- 后续语义搜索可以复用 chunk。

## 7. indexer 模块

职责：

- 对文档片段进行分词。
- 构建关键词倒排索引。
- 保存词频等排序信息。

建议接口：

```rust
pub struct InvertedIndex {
    // fields decided by implementation
}

impl InvertedIndex {
    pub fn build(chunks: &[DocumentChunk]) -> Self;
    pub fn lookup(&self, term: &str) -> Vec<String>;
}
```

要求：

- 支持中文和英文基础查询。
- 至少记录关键词到 chunk ID 的映射。
- 可扩展 TF-IDF 或 BM25。

## 8. search 模块

职责：

- 接收查询字符串。
- 从索引中查找候选片段。
- 计算相关度分数。
- 返回排序后的搜索结果。

建议接口：

```rust
pub fn search(
    index: &InvertedIndex,
    chunks: &[DocumentChunk],
    query: &str,
    top_k: usize,
) -> AppResult<Vec<SearchResult>>;
```

要求：

- 空查询应返回错误。
- 无结果时返回空列表，不应崩溃。
- 排序逻辑需要可测试。

## 9. storage 模块

职责：

- 保存索引缓存。
- 加载索引缓存。

建议接口：

```rust
pub fn save_index(path: &Path, index: &InvertedIndex, chunks: &[DocumentChunk]) -> AppResult<()>;
pub fn load_index(path: &Path) -> AppResult<(InvertedIndex, Vec<DocumentChunk>)>;
```

要求：

- 优先使用 `serde_json`。
- 缓存损坏时返回清晰错误。
- 不把缓存文件作为必须提交内容。

## 10. semantic 模块

职责：

- 提供可选语义搜索或 AI 问答增强。
- 在 AI 不可用时不影响关键词搜索。

建议接口：

```rust
pub trait SemanticEngine {
    fn rank(&self, query: &str, chunks: &[DocumentChunk]) -> AppResult<Vec<(String, f64)>>;
    fn answer(&self, question: &str, contexts: &[DocumentChunk]) -> AppResult<String>;
}
```

要求：

- API Key 从环境变量读取。
- 不在代码中硬编码密钥。
- 网络失败时返回可解释错误。
- `ask` 命令至少能回退到展示相关片段。

## 11. cli 模块

职责：

- 解析命令行参数。
- 调用对应模块。
- 输出用户可读结果。

建议命令：

```bash
cargo run -- index <path>
cargo run -- search <query> --top 5
cargo run -- ask <question>
```

要求：

- 参数错误时给出帮助信息。
- 搜索结果包含文件路径、片段和分数。
- 输出适合录制演示视频。

## 12. 修改接口流程

如果需要修改公共接口：

1. 在小组群说明修改原因。
2. 同步影响到的模块负责人。
3. 修改 `docs/接口设计.md`。
4. 更新相关测试。
5. 在提交信息中说明接口变化。
