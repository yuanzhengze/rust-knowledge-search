# 测试计划

本文档用于规划项目测试范围，确保最终项目满足课程要求中的单元测试、关键功能测试和工程规范要求。

## 1. 测试目标

测试需要保证：

- 项目可以稳定编译和运行。
- 核心模块具备单元测试。
- 主要命令具备集成测试或手动验证记录。
- 异常输入不会导致程序崩溃。
- AI 功能不可用时，基础搜索仍然可用。

## 2. 测试命令

提交前建议运行：

```bash
cargo fmt
cargo clippy
cargo test
```

如果接入网络 API，还需要手动验证：

```bash
cargo run -- ask "Rust 所有权是什么？"
```

## 3. 单元测试计划

### scanner 模块

测试内容：

- 正常目录可以扫描出 Markdown/TXT 文件。
- 不支持的文件类型会被忽略。
- 隐藏文件或隐藏目录会被忽略。
- 不存在路径返回错误。
- 空目录返回空列表。

建议测试名：

```rust
scan_documents_should_find_supported_files
scan_documents_should_ignore_unsupported_files
scan_documents_should_return_error_for_missing_path
```

### parser 模块

测试内容：

- TXT 文件可以被正确读取。
- Markdown 标题可以被提取。
- Markdown 基础标记可以被清洗。
- 空文件可以被处理。
- 非 UTF-8 或读取失败时返回错误。

建议测试名：

```rust
parse_txt_should_return_document
parse_markdown_should_extract_title
parse_document_should_handle_empty_file
```

### chunker 模块

测试内容：

- 长文档可以被切分为多个 chunk。
- 短文档至少返回一个 chunk。
- chunk 保留源文件路径。
- chunk 保留起止行号。
- chunk ID 不重复。

建议测试名：

```rust
chunk_document_should_split_long_text
chunk_document_should_keep_source_path
chunk_ids_should_be_unique
```

### indexer 模块

测试内容：

- 可以根据 chunk 构建倒排索引。
- 查询存在的关键词可以返回 chunk ID。
- 查询不存在的关键词返回空结果。
- 词频统计正确。
- 中英文混合文本可被基础处理。

建议测试名：

```rust
build_index_should_record_terms
lookup_should_return_matching_chunk_ids
lookup_should_return_empty_for_unknown_term
```

### search 模块

测试内容：

- 搜索关键词返回结果。
- 搜索结果按分数排序。
- `top_k` 参数生效。
- 空查询返回错误。
- 无结果查询返回空列表。

建议测试名：

```rust
search_should_return_ranked_results
search_should_respect_top_k
search_should_reject_empty_query
```

### storage 模块

测试内容：

- 索引可以保存到 JSON 文件。
- 索引可以从 JSON 文件加载。
- 缓存文件不存在时返回错误。
- 缓存文件损坏时返回错误。

建议测试名：

```rust
save_and_load_index_should_preserve_data
load_index_should_fail_for_missing_file
load_index_should_fail_for_invalid_json
```

### semantic 模块

测试内容：

- 没有 API Key 时返回可解释错误或降级。
- 语义排序接口可被 mock。
- `ask` 命令可以在 AI 失败时返回检索片段。

建议测试名：

```rust
semantic_engine_should_fail_without_api_key
ask_should_fallback_to_search_results
```

## 4. 集成测试计划

使用 `examples/sample_notes/` 作为示例资料目录。

### 完整索引流程

命令：

```bash
cargo run -- index ./examples/sample_notes
```

预期结果：

- 程序成功扫描示例目录。
- 成功解析 Markdown/TXT 文档。
- 成功生成索引缓存。
- 输出索引统计信息。

### 关键词搜索流程

命令：

```bash
cargo run -- search "Rust"
cargo run -- search "所有权" --top 5
```

预期结果：

- 返回相关文档路径。
- 返回相关片段。
- 返回分数或排序信息。
- 无 panic 或异常退出。

### 自然语言问答流程

命令：

```bash
cargo run -- ask "Rust 的所有权解决了什么问题？"
```

预期结果：

- 如果 AI 可用，返回简短回答和引用片段。
- 如果 AI 不可用，返回相关文档片段并说明无法生成 AI 回答。

## 5. 手动测试计划

手动测试场景：

- 中文关键词搜索。
- 英文关键词搜索。
- 中英文混合搜索。
- 空查询。
- 不存在目录。
- 空目录。
- 大文件。
- 文件名包含空格。
- 没有 API Key 的 `ask` 命令。
- 网络失败时的 `ask` 命令。

## 6. 验收标准

最终提交前应满足：

- `cargo build` 成功。
- `cargo fmt` 已执行。
- `cargo clippy` 尽量无明显警告。
- `cargo test` 中关键测试通过。
- 示例目录可以完成索引。
- 示例关键词可以搜索出结果。
- README 中说明了如何运行测试。
