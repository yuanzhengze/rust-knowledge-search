## 2026-06-02：完成索引与搜索模块（提前覆盖 Day 5 / Day 6）

已完成：

- 实现 `indexer::InvertedIndex::build` 与 `lookup`：
  - 中英文混合分词（ASCII 字母数字 + `_` 累积成 token 转小写；CJK Unified Ideographs U+4E00–U+9FFF 按字符 unigram；其他字符作为分隔符），零依赖。
  - `terms` 单 chunk 内去重，`term_freq` 累计真实出现次数；标题与正文同样参与分词。
  - `lookup` 用与 `build` 一致的 `normalize_query_term` 正则化输入；未命中、空串、纯标点都返回空 `Vec`，不报错。
  - 暴露 `pub(crate) fn tokenize` 给 `search` 复用，对外公共 API 不变。
- 实现 `search::search`：
  - 空查询、仅空白、仅标点、`top_k == 0` 全部返回 `AppError::Index`。
  - 词频累加评分；同分按 `chunk_id` 升序稳定排序；`SearchResult.snippet` 按 char 截断（不破坏 UTF-8），上限 120 字符；`matched_terms` 用 `BTreeSet` 去重并排序。
  - `chunks` 切片与 `index` 不一致时（chunk_id 缺失）安全跳过，不 panic。
- 单测覆盖 `docs/测试计划.md` §3.4 / §3.5 全部用例：
  - indexer 9 个：分词正向 / 标点剪枝 / `terms` 构建 / lookup 命中 / lookup 不命中 / 词频统计 / 标题参与分词 / 空 chunk 跳过 / serde 双向序列化。

## 2026-06-02：完成解析与切片模块（覆盖 Day 3 / Day 4，端到端打通）

已完成：

- 实现 `parser::parse_document`：
  - `fs::read` + `String::from_utf8` 校验 UTF-8，失败映射 `AppError::Parse` 并附带首个非法字节的偏移。
  - 按扩展名分流：`.md`（大小写不敏感）走 Markdown 清洗 + H1 标题提取，`.txt` 直读 + 首个非空白行作 title。
  - Markdown 清洗用 `regex` crate 做 inline 替换：image / link / bold（`**` `__`）/ italic（`*` `_`）/ inline code 全部去标记保留文本；HTML 注释剔除；行首前导（heading / list / blockquote）剥掉；代码围栏行替换为空行，围栏内代码内容保留以便索引。
  - **关键不变量：清洗后行数与原文一致**，让 chunker 行号对齐原始文件。
- 实现 `chunker::chunk_document`：
  - 按行累积 + `max_chars` 阈值切片；`max_chars == 0` 安全 clamp 到 1，避免死循环。
  - chunk ID 形式 `<file_path>#L<start>-L<end>`，1-based 行号。
  - 空文档至少返回 1 个空 chunk；单行超过 `max_chars` 时该行单独成片，不在行内继续切。
  - chunks 之间行号连续不重叠：`chunks[i+1].start_line == chunks[i].end_line + 1`。
- 单测覆盖 `docs/测试计划.md` §3.2 / §3.3 全部用例，外加端到端用例：
  - parser 11 个：TXT / 跳过空白行取 title / Markdown 标题 / inline 标记剥除 / list+blockquote 剥除 / 行数保持 / 围栏内代码保留 / 空文件 / 非 UTF-8 错误 / 真实样例 md / 真实样例 txt。
  - chunker 11 个：长文切多片 / 短文 1 片 / 空文 1 片 / 路径与标题传递 / ID 唯一 / ID 格式 / 行号连续不重叠 / `max_chars=0` 不死循环 / 单行超长安全处理 / 真实样例端到端 / **scanner→parser→chunker→indexer→search 完整流水线跑 sample_notes**（Rust + 所有权 都能命中）。
- 本地校验：`cargo build` ✓ / `cargo test` 49 passed 0 failed（集成测试 3 条仍 `#[ignore]`，等 cli/storage）/ `cargo clippy --all-targets -- -D warnings` 0 warning ✓ / `cargo fmt --check` ✓。

遇到的问题：

- **regex `\s` 默认包含 `\n`，会跨行吞换行符**：最初 `(?m)^\s*#{1,6}\s+` 匹配 `\n## ` 时把前一行的换行也吃掉，行数从 12 变成 10，破坏了"清洗后行数不变"的不变量。修复方法是把所有行首前导 regex 中的 `\s` 换成 `[ \t]`，限制为水平空白。
- **chunker 测试阈值算错了**：第一版 `chunk_id_format_*` 测试用 `max_chars=6` 期望 "abc\ndef" 合并到一个 chunk，但实际 7 字符（含换行）已超阈值，每行各成一片。把阈值调到 `8` 后符合预期。这也间接确认了切片边界条件正确。
- 没有引入 `tempfile` 依赖；parser 测试沿用 scanner 那边的零依赖 `TempDir` 风格，新建一份在 parser 单测内（避免跨模块共享 test util 牵动公共代码）。
