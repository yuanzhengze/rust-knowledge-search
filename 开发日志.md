## 2026-06-02：完成文件扫描模块

已完成：

- 实现 `scanner::scan_documents`，使用 `walkdir` 递归遍历目录，过滤 `.md` / `.txt`（扩展名大小写不敏感），跳过隐藏文件、隐藏目录与 `target/`。
- 路径不存在或非目录时返回 `AppError::InvalidPath`；`walkdir::Error` 中的 IO 错误透传为 `AppError::Io`，目录权限问题不再被静默吞掉。
- 在 `scanner.rs` 同文件下补 8 个单测，覆盖 `docs/测试计划.md` §3.1 全部用例：示例目录正向扫描、不支持扩展名过滤、缺失路径报错、传入文件路径报错、空目录返空、隐藏条目剪枝、`target/` 剪枝、元数据字段完整性。
- 单测内置零依赖临时目录工具（基于进程 PID + 纳秒时间戳 + 原子计数器），不引入 `tempfile` 依赖。
- 本地校验：`cargo build` / `cargo test`（scanner 8/8 通过，集成测试保持 `#[ignore]`）/ `cargo clippy --all-targets -- -D warnings` / `cargo fmt --check` 全部通过。

遇到的问题：

- `walkdir::Error::into_io_error` 返回 `Option<io::Error>`，需要兜底以避免在拿不到 IO 错误时丢失上下文，最终在元数据读取失败路径上回退到 `AppError::InvalidPath("metadata read failed")`。
- `filter_entry` 在 `depth == 0` 时也会触发，若 `root` 自身以 `.` 开头会被整体剪掉，因此 `is_hidden` 加了 `depth() > 0` 守卫。
