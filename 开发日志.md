## 2026-06-02：完成文件扫描模块

已完成：

- 实现 `scanner::scan_documents`，使用 `walkdir` 递归遍历目录，过滤 `.md` / `.txt`（扩展名大小写不敏感），跳过隐藏文件、隐藏目录与 `target/`。
- 路径不存在或非目录时返回 `AppError::InvalidPath`；`walkdir::Error` 中的 IO 错误透传为 `AppError::Io`，目录权限问题不再被静默吞掉。
- 在 `scanner.rs` 同文件下补 8 个单测，覆盖 `docs/测试计划.md` §3.1 全部用例：示例目录正向扫描、不支持扩展名过滤、缺失路径报错、传入文件路径报错、空目录返空、隐藏条目剪枝、`target/` 剪枝、元数据字段完整性。
- 单测内置零依赖临时目录工具（基于进程 PID + 纳秒时间戳 + 原子计数器），不引入 `tempfile` 依赖。
- 本地校验：`cargo build` / `cargo test`（scanner 8/8 通过，集成测试保持 `#[ignore]`）/ `cargo clippy --all-targets -- -D warnings` / `cargo fmt --check` 全部通过。

遇到的问题：

- `walkdir::Error::into_io_error` 返回 `Option<io::Error>`，需要兜底以避免在拿不到 IO 错误时丢失上下文，最终在元数据读取失败路径上回退到 `AppError::InvalidPath("metadata read failed")`。
- `filter_entry` 在 `depth == 0` 时也会触发，若 `root` 自身以 `.` 开头会被整体剪掉，因此 `is_hidden` 加了 `depth() > 0` 守卫。

## 2026-06-02：完成索引与搜索模块（提前覆盖 Day 5 / Day 6）

已完成：

- 实现 `indexer::InvertedIndex::build` 与 `lookup`：
  - 中英文混合分词（ASCII 字母数字 + `_` 累积成 token 转小写；CJK Unified Ideographs U+4E00–U+9FFF 按字符 unigram；其他字符作为分隔符），零依赖。
  - `terms` 单 chunk 内去重，`term_freq` 累计真实出现次数；标题与正文同样参与分词。
  - `lookup` 用与 `build` 一致的 `normalize_query_term` 正则化输入；未命中、空串、纯标点都返回空 `Vec`，不报错。
  - 暴露 `pub(crate) fn tokenize` 给 `search` 复用，对外公共 API 不变。
- 实现 `search::search`：
  - 空查询、仅空白、仅标点、`top_k == 0` 全部返回 `AppError::Index`。
  - 词频累加评分；同分按 `chunk_id` 升序稳定排序；`SearchResult.snippet` 按 char 截断（不破坏 UTF-8），上限 120 字符；`matched_terms` 用 `BTreeSet` 去重并排序。
  - `chunks` 切片与 `index` 不一致时（chunk_id 缺失）安全跳过，不 panic。
- 单测覆盖 `docs/测试计划.md` §3.4 / §3.5 全部用例：
  - indexer 9 个：分词正向 / 标点剪枝 / `terms` 构建 / lookup 命中 / lookup 不命中 / 词频统计 / 标题参与分词 / 空 chunk 跳过 / serde 双向序列化。
  - search 10 个：排序 / `top_k` 截断 / 空查询报错 / 纯标点报错 / `top_k=0` 报错 / 无结果空 Vec / 多词累加 / 中文查询 / snippet 长度上限 / chunks 切片不一致。
- 本地校验：`cargo build` ✓ / `cargo test` 27 passed 0 failed（集成测试保持 `#[ignore]`）/ `cargo clippy --all-targets -- -D warnings` 0 warning ✓ / `cargo fmt --check` ✓。
- 未触碰其他人模块；未修改 `Cargo.toml` 与 `docs/接口设计.md`（`InvertedIndex` 字段早已声明 `pub`，`tokenize` 是 `pub(crate)` 不属于对外公共面）。

遇到的问题：

- `char` 在 `matches!` 宏中无法直接用十六进制范围匹配，需要 `ch as u32` 后再做 range pattern。
- `cargo fmt` 会把短的 `vec![...]` 折成单行，与人手写的多行风格不一致；以工具输出为准，已统一通过。
- search 与 indexer 的分词必须严格一致，否则 query 命中不到索引；最终把 `tokenize` 作为 `pub(crate)` 共享，避免在两处各写一份产生漂移。
