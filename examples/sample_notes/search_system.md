# 本地知识库搜索系统

本地知识库搜索系统通常包括文件扫描、文本解析、文档切片、索引构建和结果排序几个阶段。

## 文件扫描

文件扫描模块负责遍历用户指定的目录，识别可以处理的文档格式，例如 Markdown 和 TXT。扫描阶段只负责发现文件和读取元数据，不应该处理复杂的文本逻辑。

## 文档解析

文档解析模块负责读取文件内容，并将不同格式的文档转换为统一的文本结构。Markdown 文档可以提取标题，并清理部分标记。

## 文档切片

长文档不适合直接作为搜索结果展示，因此需要切分为较短片段。每个片段应保留来源文件、标题和行号范围。

## 倒排索引

倒排索引记录关键词到文档片段的映射关系。搜索时可以先根据关键词找到候选片段，再根据词频、TF-IDF 或 BM25 计算相关度。

## 语义增强

语义搜索可以使用 embedding 表示问题和文档片段，再通过向量相似度找到相关内容。在本项目中，语义增强是可选模块，基础搜索功能不依赖 AI。
