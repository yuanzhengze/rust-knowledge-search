# Rust 所有权笔记

Rust 的所有权系统用于在没有垃圾回收器的情况下保证内存安全。每个值都有一个所有者，当所有者离开作用域时，值会被自动释放。

## 所有权规则

Rust 的所有权有三条基本规则：

- Rust 中的每个值都有一个所有者。
- 同一时间一个值只能有一个所有者。
- 当所有者离开作用域时，该值会被丢弃。

## Move

当一个值被赋给另一个变量时，如果类型没有实现 `Copy`，所有权会发生移动。移动之后，原变量不能继续使用。

## Borrow

借用允许函数或代码块临时访问一个值，而不取得所有权。不可变借用可以同时存在多个，可变借用同一时间只能存在一个。

## 为什么重要

所有权机制让 Rust 能够在编译期发现悬垂引用、重复释放和数据竞争等问题。这也是 Rust 适合系统工具、并发程序和高可靠服务的原因之一。
