Rust 错误处理笔记

Rust 通常使用 Result<T, E> 表示可能失败的操作。文件读取、目录扫描、网络请求、JSON 解析等操作都应该返回 Result，而不是直接 panic。

在本地知识库搜索系统中，常见错误包括：

1. 用户输入的目录不存在。
2. 文件没有读取权限。
3. 文档不是合法的 UTF-8 文本。
4. 索引缓存文件损坏。
5. AI API 没有配置密钥或网络请求失败。

良好的错误处理应该给出清晰提示，并尽量让程序在可恢复场景下降级运行。例如，AI 问答失败时，系统仍然可以返回关键词搜索得到的相关文档片段。

为了便于维护，可以在 error.rs 中定义统一的错误类型，并在各模块中返回 AppResult<T>。
